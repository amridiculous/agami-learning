# Reviews Folder

This folder holds all phase outputs from the agency workflow.
Files are generated in phase order and never deleted between phases.

## File Index

| File | Generated by | Phase |
|---|---|---|
| `design-spec.md` | design-agent | Phase 1 |
| `dev-notes.md` | developer-agent | Phase 2 |
| `design-qa-report.md` | design-qa-agent | Phase 3 |
| `code-review-report.md` | code-reviewer-agent | Phase 4 |
| `qa-report.md` | qa-test-agent | Phase 5 |
| `final-review.md` | orchestrator | Phase 6 |

## Workflow

```
Phase 1 → design-spec.md       (CHECKPOINT — human approves before build)
Phase 2 → dev-notes.md         (developer declares deviations)
Phase 3 → design-qa-report.md  (loop back to developer if FAIL)
Phase 4 → code-review-report.md (loop back to developer if FAIL)
Phase 5 → qa-report.md         (loop back to developer if FAIL)
Phase 6 → final-review.md      (CHECKPOINT — human final sign-off)
```

## Verdicts
- Any phase that returns FAIL sends the report back to the developer-agent.
- Loops are capped at 3. On the 4th failure, escalate to the human.
- PASS in all phases → proceed to Phase 6 Final Review.
